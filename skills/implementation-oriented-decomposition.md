# Implementation-Oriented Decomposition

## When to use

Use this skill when the user wants a code change, architecture change, or implementation plan.

## SOP

1. Identify the user-visible behavior and acceptance criteria.
2. Locate affected modules, boundaries, and data flow.
3. Choose the smallest coherent implementation slice.
4. Define interfaces, schemas, or contracts before wiring UI.
5. Implement incrementally with focused validation.
6. Run relevant checks and summarize remaining risks.

## Evidence guidance

Use repository code, tests, local execution, compiler output, and issue acceptance criteria.

## Trace note

Show the module boundary and validation strategy.
