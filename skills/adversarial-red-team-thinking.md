# Adversarial / Red-Team Thinking

## When to use

Use this skill when failure modes, abuse paths, or regressions matter.

## SOP

1. Identify the asset, claim, workflow, or system under risk.
2. List plausible failure modes and misuse cases.
3. Check which failures are likely versus merely possible.
4. Identify safeguards, validation, and monitoring.
5. Recommend fixes that reduce real risk without overcomplicating the system.
6. State residual risks after mitigation.

## Evidence guidance

Prefer reproducible failures, concrete attack surfaces, logs, tests, or direct system behavior.

## Trace note

Show the key failure mode considered and the residual risk.
