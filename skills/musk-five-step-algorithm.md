# Musk Five-Step Algorithm

## When to use

Use this skill when the user asks how to simplify, ship faster, remove waste, or debug an overcomplicated plan.

## SOP

1. Question each requirement and identify the owner of that requirement.
2. Delete any step, feature, dependency, or process that is not clearly needed.
3. Simplify and optimize the remaining path.
4. Accelerate feedback loops and reduce cycle time.
5. Automate only after the process is proven necessary and stable.
6. Call out any part that should not be automated yet.

## Evidence guidance

Prefer direct workflow evidence, constraints, costs, and failure modes over abstract process advice.
